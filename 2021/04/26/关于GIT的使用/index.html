Git与Github

发表于 2021-04-26 更新于 2021-04-28 分类于 git

关于GIT的使用

使用GIT上传到GITHUB的仓库中是一种常见的操作，总结一下使用的代码

操作平台：WINDOWS10

1.SSH

GitHub的传输分为SSH，HTTP两种（CLI暂无使用）HTTP的使用貌似比较简单，等以后有机会在进行讨论，SSH必须是项目的管理者或者拥有者才可使用。无需在每次访问都提供用户名。

1.SSH的建立

在一台新设备需要创建一个SSH KEY才可以进行Clone和Push

先在本地创建SSH KEY 输入

ssh-keygen -t rsa -C "自己的邮箱"

该邮箱为上传之后的上传者信息。

第二步会要求你输入密码，可以选择。

之后在C盘里用户文件夹有一个名为.ssh的文件夹，会出现一个名为id_rsa的文件，打开以后将内容复制

打开Github，在左上角的头像点开Settings，点击SSH and GPG Keys 点击New SSH Key，TITLE 随便写个名字就完事了，将内容复制到下面即可。

然后输入ssh -T git@github.com回车以后是否看到You’ve successfully authenticated …….这就表示成功。

2．配置ＧＩＴ

在需要上传的文件夹中建立一个.git文件，命令为　git init　（意思为ｇｉｔ初始化）

3.从本地上传到Github

==注：这里是第一次使用，新建的是空仓库！==

如果是将文件上传到已有代码的仓库需要一些后面会标注的注意事项

在上传之前肯定要设置username和email

git config --global user.name "your name"

git config --global user.email "your_email@....com"

就是输入你的github账号 具体用户名是什么用户名后面再进行研究。

输入

git add .

或者将.改为*或者–all 意思为全部上传，或者添加单一程序仅需加上名字即可，此步骤为将文件上传到暂存区。

之后 输入git commit -m "备注"为你提交代码添加备注，不可以缺少该步骤，因为github的上传必须拥有备注。

你也可以创建REDME.MD

之后将其上传到Github上，输入指令

git remote add origin 你仓库的SSH

git push -u origin master此为将代码上传到Master分支上，第一次需要-u

成功上传

5.将代码上传到已有仓库

这里需要分类讨论，

  1. 仓库只有一个分支如Master

    简单的上传

  2. 仓库有多个分支

Git中对分支的概念

分支的使用以及切换

多人合作项目的使用及操作

1.先从最简单的单分支上传

首先先要remote到远程仓库

git remote add origin <ssh>这里的origin是你给你这个ssh弄个名称写个hhh都是可以的

之后将仓库PULL到文件里

git pull origin master可以更改

你可以将它理解为是clone到本地文件夹

之后就是

git add .

git commit -m '备注'

随后将本地仓库的代码上传到远程仓库中

git push origin master

如果本地仓库与远程仓库不一致的话是上传不了的

2.仓库拥有多个分支

首先Clone的意思是在本地建立一个一模一样的的版本库到本地

而pull是从远程仓库获取一个branch分支并更新到本地，再与本地的指定分支进行合并。

根据步骤第一步是remote到远程仓库这一点没什么好说的本地与远程的分支都不会出现

然后就是pull，假设我有一个分支叫做test,将test分支中的文件PULL到本地

git pull origin test

下面提示中会出现

FTECH_HEAD的意思是跟踪刚刚从远程存储库中获取的内容

git pull是由两个过程合并而成：

git fetch 和git merge

前者是从远程获取分支，于是FETCH_HEAD指向此分支的尖端，就是最新的文件，是跟踪。

git merge 合并FETCH_HEAD到当前分支

使用

git branch

分支查询以后会发现多了一个master的分支，这个是你本地的仓库分支

使用

git branch -a

查询所有分支以后发现多了一个名为remoter/origin/test的分支，这就是你的远程分支，本地会出现一个master的分支

之后就是进行add，commit这些

在使用push之前先看push的语句：

git push <远程主机名> <本地分支名>:<远程分支名>

其中远程主机名就是你在remote时定义的名称例如origin

在本地分支名与远程分支名相同的时候，则可以省略冒号和后尾

最后我们进行上传

git push origin master:test

成功上传

你也可以写成

git push origin head:test

HEAD分支你可以理解为现在所指向的分支，你可以使用git checkout 分支名来进行切换

如果你要创建分支使用 git branch 分支名进行创建本地分支

3.入门多人合作及合并

理解以上以后不难理解入门多人操作的概念，就是好几个人进行不同的分支开发然后然后在本地进行合并。

  1. git merge 分支先将分支内容合并到本地分支
  2. git pull 防止其他人在你上传的时候进行提交
  3. git push origin ...:master进行合并
